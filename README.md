# ChatBI — 用自然语言与你的数据对话

ChatBI 是一款面向中小企业的智能数据分析工具。上传 Excel 或 CSV 文件，系统自动生成可视化仪表盘，并支持用自然语言提问，让没有数据分析背景的用户也能快速读懂业务数据。

---

## 核心功能

- **一键上传，自动出图** — 上传 `.xlsx`、`.xls`、`.csv` 文件后，系统自动分析数据结构，生成图表与业务洞察
- **自然语言问答** — 直接用中文提问，如"三月哪个产品销量最高？"，系统计算后以文字和图表形式作答
- **数字可信赖** — 所有答案基于真实计算结果，AI 先算后说，杜绝数字幻觉
- **多文件支持** — 可同时上传多个文件，跨表分析

| 角色 | 典型问题 |
|---|---|
| 老板/店主 | 今天赚了多少？哪些产品最好卖？ |
| 仓库/采购 | 哪些商品快过期？库存不足的有哪些？ |

不需要懂 Excel 公式，不需要会写 SQL，用说话的方式就能获取业务数据答案。

## 使用流程

1. **上传文件** — 拖拽或点击上传 Excel / CSV 文件（单文件最大 5MB）
2. **查看仪表盘** — 系统自动生成图表与关键指标卡片
3. **开始提问** — 在底部输入框用自然语言提问，获取实时分析结果

---

## 限制说明（当前版本）

- 无登录认证，会话数据不持久化（30 分钟无操作后自动清除）
- 最多支持 50 个并发会话
- 单文件大小限制 5MB
- 支持格式：`.xlsx`、`.xls`、`.csv`

## 答案是如何生成的

ChatBI 的问答系统采用**两次 LLM 调用**机制，确保每一个数字都来自真实计算，而非 AI 猜测。

### 计算型问题（如"三月销售额是多少？"）

```
用户提问
  → 第 1 次调用：DeepSeek 理解问题，生成结构化分析计划（JSON）
  → AnalysisEngine 在真实数据上执行计划，得到真实数字
  → 第 2 次调用：DeepSeek 将真实计算结果转化为自然语言回答
  → 返回：答案文字 + 图表
```

### 对话型问题（如"这个趋势说明什么？"）

```
用户提问
  → 第 1 次调用：DeepSeek 识别为对话型，无需计算
  → 第 2 次调用：基于数据摘要与历史对话给出解读
  → 返回：答案文字，无图表
```

### 核心设计原则

**AI 先算后说** — LLM 在看到真实计算结果之前，绝不生成含数字的答案。这从根本上杜绝了数字幻觉。

| | 单次调用 | ChatBI 的两次调用 |
|---|---|---|
| LLM 依据 | 列名和样本值（猜测） | 真实计算结果 |
| 数字可信度 | 可能是假的 | 100% 来自真实数据 |

## 后端架构

```
POST /api/upload   文件上传 → 解析 → 数据分析 → 自动生成仪表盘
POST /api/chat     用户提问 → 分类 → 计算 → 生成答案
GET  /api/session  获取当前会话状态
```

- **FileParser** — 解析 xlsx / xls / csv，转为 DataFrame
- **DataProfiler** — 分析数据结构（列类型、统计值、样本），生成 SheetProfile 供 LLM 使用
- **AnalysisEngine** — 执行 LLM 生成的分析计划，在真实数据上计算，LLM 不生成可执行代码
- **LLMClient** — 封装与 DeepSeek 的交互，分离分类、计划、叙述三个调用

---

## 项目背景

基于对真实用户反馈的研究，中小企业主在使用进销存系统时面临五大痛点：报表缺失、字段不足、维度受限、缺乏可视化、数据无法转化为业务洞察。ChatBI 旨在用 AI 一次性解决这些问题，让用户通过自然语言获取个性化的业务报表，而无需依赖固定报表模板。
